# Società Agricola Farina 2.0

Marketing website for [www.cucurbitacee.com](https://www.cucurbitacee.com), built with Vite, React, Tailwind, and deployed to GitHub Pages.

## What This Repo Contains

- Multi-page static site for the farm business
- Product pages for angurie, meloni, zucche, and legna da ardere
- Contact, gallery, company, privacy, and cookie-policy pages
- GitHub Pages deploy workflow
- Generated sitemap for SEO

## Local Development

```bash
npm ci
npm run dev
```

Useful commands:

```bash
npm run lint
npm run test
npm run build
npm run sync:sitemap
npm run sync:images
```

## Deployment

Deploys are handled by [deploy.yml](./.github/workflows/deploy.yml) on pushes to `main`.

GitHub Pages should publish the `dist/` artifact produced by the workflow, not the source files directly.

## Domain Notes

- Primary production domain: `https://www.cucurbitacee.com`
- Custom-domain file: [public/CNAME](./public/CNAME)
- Site URL constant used by SEO tags: [src/lib/site.ts](./src/lib/site.ts)

If the production domain changes, update:

1. `public/CNAME`
2. `src/lib/site.ts`
3. `public/robots.txt`

## Content Maintenance

- Route entries used for the Vite multi-page build and sitemap generation live in [scripts/site-pages.mjs](./scripts/site-pages.mjs).
- The sitemap is generated by [scripts/generate-sitemap.mjs](./scripts/generate-sitemap.mjs) before each build.
- When adding a new top-level page, add it once to `scripts/site-pages.mjs` and then create the matching page component and HTML entry file.

## Media And External Embeds

- Image assets live in `src/assets/`
- Modern `webp` and `avif` variants are generated into `src/assets/generated/` by [scripts/generate-modern-images.mjs](./scripts/generate-modern-images.mjs) before `dev`, `test`, and `build`
- Keep the original source photos in `src/assets/`; the site serves the optimized variants automatically through [src/components/SiteImage.tsx](./src/components/SiteImage.tsx)
- The gallery page includes a Facebook page embed that loads only after user interaction for privacy/cookie reasons
- The contact page uses a keyless Google Maps embed to avoid exposing an API key in the frontend
- The contact page submits through FormSubmit, so the destination inbox must confirm the first live submission once before messages start flowing normally
